Otoroshi

The Otoroshi is a large hairy monster that tends to lurk on the top of the torii gate in front of Shinto shrines. It's a hostile creature, but also said to be the guardian of the shrine and is said to leap down from the top of the gate to devour those who approach the shrine for only self-serving purposes.

Otoroshi is also a modern http reverse proxy with a thin layer of api management written in Scala and developped by the MAIF OSS team that can handle all the calls to and between your microservices without service locator and let you change configuration dynamicaly at runtime.

  • Easy to configure

    No config files, gorgeous admin UI, slick admin API. Live configuration at runtime

  • Low latency

    Built with Scala and reactive programming. Designed, built and benchmarked for high-performances

  • Simple UI and/or API driven
  • Event-Driven
  • Public or Private apps
  • Can proxy WebSockets
  • Can Proxy (almost ;-) anything
  • Optional additional headers for integration
  • Optional U2F Authentication
  • Configurable per-service circuit breaker
  • Global or per-service throttling and IP filtering
  • Optional round-robin load balancing
  • Advanced Clever-Cloud integration
  • Services heatlhcheck
  • Multi-datastores support
    • Redis
    • In-memory
    • Cassandra
    • MongoDB (TODO)
    • PostgreSQL (TODO)

Made with love and much more...
